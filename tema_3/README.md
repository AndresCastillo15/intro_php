#  Guía Maestra: Funciones en PHP

Esta guía está diseñada para ser clara, profesional e incluye desde los conceptos más básicos hasta el tipado de datos moderno. Aprenderás a crear código modular, reutilizable y eficiente.

---

##  ¿Qué incluye esta guía?

### 1. Declaración Básica
Cómo crear y ejecutar tu primera función. Aprenderás la sintaxis fundamental para "instalar" tu bloque de código y cómo invocarlo cuando lo necesites.

### 2. Parámetros
Cómo pasarle datos a la función para hacerla dinámica. Los parámetros actúan como los **ingredientes** que la función procesará para dar un resultado distinto según lo que reciba.

### 3. Retorno (return)
La forma de "sacar" un resultado de la función. Es el acto de servir el producto final en un vaso (variable) para que pueda ser utilizado en otras partes del programa.

### 4. Valores por Defecto
Cómo evitar errores si falta un dato. Aprenderás a definir valores predeterminados para los parámetros, asegurando que la función trabaje incluso si no se le envían todos los ingredientes.

### 5. Tipado Estricto
Cómo asegurar que tu código sea más robusto definiendo tipos de datos (`int`, `string`, `bool`, etc.). Esto ayuda a prevenir errores y hace que el código sea más profesional y fácil de entender.

### 6. Funciones Flecha (Arrow Functions)
La sintaxis moderna y reducida introducida en PHP 7.4+. Ideal para funciones de una sola línea, permitiendo escribir código más limpio y directo.

### 7. Ámbito (Scope)
La explicación de por qué las variables de afuera no se ven adentro y viceversa. Entenderás cómo gestionar la visibilidad de los datos para evitar conflictos en tu aplicación.

---

## La Analogía de la Licuadora

Para entender las funciones, recuerda siempre este concepto:
- **La Función:** Es la licuadora (la máquina con una tarea específica).
- **Parámetros:** Son las frutas y líquidos (los ingredientes que ingresas).
- **Cuerpo de la función:** Es el proceso de licuado (la lógica interna).
- **Return:** Es el batido servido en el vaso (el resultado que obtienes).
